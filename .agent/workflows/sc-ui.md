# UI Workflow

Use this workflow for any UI build, redesign, or UI quality review.

## Steps

1. Read `.agent/context/workflows/sc-ui.contract.md` and `.agent/context/skills/interface-design.contract.md` for the compact runtime path.
2. Load `skills/interface-design/SKILL.md` when following the full UI procedure.
3. Read `.agent/rules/project-config.md` to identify frontend stack, styling, and component library.
4. Generate or read the project design system before implementation.
5. Run targeted domain searches only for the UI risks in scope: `web`, `app`, `ux`, `chart`, `typography`, `icons`, or stack-specific guidance.
6. Read search results only; do not preload interface CSV files into model context.
7. Implement with existing components and project conventions.
8. Verify responsive behavior, accessibility, text overflow, interaction states, loading/empty/error states, and stack-specific risks.

## Output

- UI changes grounded in the design guidance database.
- Verification evidence for layout, accessibility, and interaction quality.
