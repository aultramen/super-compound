# Status Workflow

Use this at the start of a session or when orientation is needed.

## Steps

1. Check for `.continue-here.md`.
2. Read `docs/STATE.md` and `docs/progress.md` when present.
3. Check Git status when inside a Git repo.
4. Summarize current position, completed work, remaining work, blockers, and verification status.
5. Recommend the next workflow: `explore`, `prd`, `plan`, `work`, `debug`, `review`, `audit`, `compound`, or `pause`.

## Output

- Short dashboard.
- Recommended next action.
