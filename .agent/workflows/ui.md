# UI Workflow

Use this workflow for any UI build, redesign, or UI quality review.

## Steps

1. Load `skills/interface-design/SKILL.md`.
2. Read `.agent/rules/project-config.md` to identify frontend stack, styling, and component library.
3. Generate or read the project design system before implementation.
4. Run targeted domain searches only for the UI risks in scope: `web`, `app`, `ux`, `chart`, `typography`, `icons`, or stack-specific guidance.
5. Implement with existing components and project conventions.
6. Verify responsive behavior, accessibility, text overflow, interaction states, loading/empty/error states, and stack-specific risks.

## Output

- UI changes grounded in the design guidance database.
- Verification evidence for layout, accessibility, and interaction quality.
