# Documentation Rules

- Keep startup files short: `AGENTS.md` for shared guidance, `CLAUDE.md` for Claude-only memory behavior.
- Keep long standards and examples in normal docs such as `docs/engineering-standards.md`.
- Prefer concrete paths, commands, and expected outcomes over vague guidance.
- Preserve existing headings and command names unless a rename is part of the requested change.
- For docs-only changes, verify by reading the changed files and checking that referenced paths exist when practical.
